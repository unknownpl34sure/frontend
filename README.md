# Studwork — фронтенд

Биржа фриланса для студентов. React + Vite + Tailwind CSS v4, работает поверх
FastAPI-бэкенда из папки [`backend`](https://github.com/freelance-bibi/backend)).

## Стек

- **React 19** + **React Router 7**
- **Vite 8**
- **Tailwind CSS v4** (через `@tailwindcss/vite`)
- **axios** — HTTP-клиент
- **lucide-react** — иконки

## Быстрый старт

```bash
npm install
npm run dev
```

Приложение поднимется на `http://localhost:5173` (если порт занят — Vite выберет
следующий свободный).

Сначала запустите бэкенд (см. [`backend`](https://github.com/freelance-bibi/backend))) —
по умолчанию он ожидается на `http://localhost:8000`.

## Конфигурация

Адрес API задаётся через переменную окружения в `.env`:

```
VITE_API_URL=http://localhost:8000
```

## Структура

```
src/
  lib/          API-клиент (api.js), форматтеры, константы
  context/      AuthContext — авторизация и текущий пользователь
  components/   UI-кит, навбар, футер, карточки, блоки профиля
  pages/        Лендинг, авторизация, объявления, чат, профиль
```

## Страницы

- `/` — лендинг
- `/login`, `/register` — авторизация
- `/tasks` — список объявлений с фильтрами (поиск, категории, цена, сортировка)
- `/tasks/:id` — страница объявления + отклик (создаёт чат)
- `/tasks/new` — размещение объявления
- `/messages`, `/messages/:chatId` — мессенджер
- `/profile` — свой профиль (навыки, портфолио, отзывы, объявления)
- `/users/:id` — публичный профиль другого пользователя

## Заметки по интеграции с бэкендом (MVP)

Ключевые эндпоинты, на которые опирается фронт:

1. **CORS** — middleware в `app/main.py` для дев-origin'ов.
2. **`GET /api/users/{id}`** — публичный профиль (нужен для авторов объявлений,
   собеседников в чате и авторов отзывов).
3. **`POST /api/kworks/{id}/order`** — «взять/заказать» объявление от лица
   покупателя. Эндпоинт ставит `client_id`, статус `in_process` и
   автоматически создаёт чат (исходный `PATCH /status` доступен только автору).
4. **WebSocket `/api/messages/ws/{chat_id}?token=<jwt>`** — живой обмен
   сообщениями в чате.

Фронтовые «костыли» MVP:

- Категории — статический список (`lib/constants.js`); фильтрация по совпадению
  ключевых слов, т.к. бэкенд не отдаёт список категорий/тегов.
- «Мои объявления» / объявления пользователя получаются фильтрацией общего
  списка `GET /api/kworks/` по `user_id` (нет отдельного эндпоинта).

Полное описание API бэкенда — в [`../backend/README.md`](../backend/README.md).
